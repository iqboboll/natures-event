## 🛠️ Manual Deployment Commands
To deploy using the unified Dockerfile, specify the target for each service:

```bash
# 1. Build & Push Backend
docker build --target backend -t gcr.io/<PROJECT_ID>/guardian-backend .
docker push gcr.io/<PROJECT_ID>/guardian-backend
gcloud run deploy guardian-backend --image gcr.io/<PROJECT_ID>/guardian-backend --region <YOUR_REGION>

# 2. Build & Push Frontend
docker build --target frontend -t gcr.io/<PROJECT_ID>/guardian-frontend .
docker push gcr.io/<PROJECT_ID>/guardian-frontend
gcloud run deploy guardian-frontend --image gcr.io/<PROJECT_ID>/guardian-frontend --region <YOUR_REGION>
```

> [!TIP]
> You can now deploy from the root directory using a single `Dockerfile`. The backend remains the default stage if no target is specified.

> [!WARNING]
> Ensure that `.env` and `.json` secret files are NOT committed to public version control (GitHub), but they MUST be present in the local directory when running the manual `gcloud deploy` command so they can be included in the build.
